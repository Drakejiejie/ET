# Python-source-code-dynamic-encryption-system

## 一、安全性
- 加密方案基于 Python 源码的 C 层 TPM 安全模块开发，私钥极难被获取，安全性极高
- 逆向成本极高，软件与设备强制绑定，不可迁移
- 远程服务器支持异常设备监控，实时把控使用风险

## 二、硬件与系统要求
- 硬件：需 TPM 硬件支持（是否支持可咨询）
- 系统：Windows 7 及以上版本
- 架构：x86 或 x64

## 三、开发环境要求
- Python 版本：3.10.9

## 四、客户部署流程
- 一键打包：快速完成程序打包，无需复杂配置
- 临时凭证机制：无权限用户无法访问您的 Python 程序，保障权限安全
- 一键注册：简化客户注册流程，降低使用门槛

## 五、基本原理
提供私有化部署的 Python 源码加密服务器，当客户请求访问 Python 程序时，服务器会动态加密目标 Python 模块，并将加密后的程序返回给客户，确保源码不泄露。

## 六、实时更新能力
### 1. 程序实时更新
因服务器动态加载并返回程序，客户每次请求获取的都是最新版本的程序，无需手动更新。

### 2. Pip 包实时更新
调用专属 API 即可实现 Pip 包的实时更新；额外提供扩展 API（如一键创建快捷方式），提升使用便捷性。
